## Hypernetworks Knowledge Distillation for Image Classification

A regular classification model is used to generate a priori class probabilities.
These probabilities are used as input for a hypernetwork to generate specialized weights for a second classification model.


## 🧪 Model Variants

We compare several model variants to study the impact of hypernetworks and conditional weight generation:

- **Regular ResNets**  
  Standard convolutional networks with fixed weights, serving as a baseline.

- **Pure Hypernetwork ResNets**  
  All convolutional and linear weights are generated by a hypernetwork conditioned on an auxiliary input \( z \):  
  \( W = \Delta W(z) \)

- **Residual Hypernetwork ResNets**  
  Combine static weights with hypernetwork-generated updates:  
  \( W = W_{\text{base}} + \alpha \cdot \Delta W(z) \)  
  This enables residual adaptation of a base network while preserving its initial structure.

- **Gated Hypernetwork ResNets**  
  Extend the residual approach with a learnable gating mechanism:  
  \( W = (1 - g) \cdot W_{\text{base}} + g \cdot \Delta W(z) \)  
  where \( g \in [0,1] \) is learned (and optionally scheduled), enabling the model to adaptively interpolate between static and dynamic weights.

- **Modulated ResNets (Feature-wise Modulation)**  
  Apply low-dimensional scaling and shifting to activations or weights:  
  \( y = \gamma(z) \cdot f(x) + \beta(z) \)

---

These variants allow us to analyze trade-offs between stability, expressiveness, and conditional adaptation, as well as the impact of normalization and scaling strategies on training dynamics.

## ⚠️ Preventing Prediction Bypass in Hypernetworks

### 🧠 Problem: Prediction Bypass

When using hypernetworks conditioned on an auxiliary signal \( z \) (e.g., predictions from another model), we observed a critical failure mode:

> **The model can learn to ignore the input image and rely almost entirely on \( z \).**

In this case, the network effectively becomes a reparameterized version of the predictor:

\[
y \approx f(z)
\]

instead of learning a meaningful combination:

\[
y = f(x, z)
\]

---

## ⚠️ Hypernetwork Shortcut Learning Issue

If performance remains unchanged when the image is removed, the model is not using visual features.

### 🔍 Observed Results

- **Pure hypernetworks (full weight generation):**  
  Accuracy remains nearly unchanged when `x = 0`  
  → Strong evidence of bypass

- **Partial hypernetworks (e.g., single hyper layer):**  
  Still capable of reconstructing predictions from `z`

- **Hypernetwork applied only at the FC layer:**  
  Retains high performance, but still shows strong dependence on `z`

These results indicate that even limited use of hypernetworks can introduce shortcut paths.

---

### 🚨 Why This Happens

Hypernetworks generate weights conditioned on `z`:

```math
W = \Delta W(z)

When `z` is highly informative (e.g., class predictions), the model can:

- Encode decision boundaries directly in generated weights  
- Use weak or constant features from `x`  
- Reconstruct predictions without meaningful feature extraction  

This creates an unintended shortcut:

z → weights → logits

### 🛠️ Ongoing Work

This behavior was not anticipated initially and emerged during experimentation.

We are currently exploring strategies to mitigate this issue and enforce meaningful use of the input `x`. These include:

- Restricting hypernetworks to later layers (e.g., classification head only)  
- Removing bias terms to avoid direct signal injection  
- Using residual or gated formulations:

```math
W = W_{\text{base}} + \Delta W(z)

```math
W = (1 - g)\, W_{\text{base}} + g\, \Delta W(z)

- Reducing hypernetwork capacity (e.g., modulation or low-rank updates)  
- Regularizing the hypernetwork (dropout in trunk, noise injection in `z`)

### 🎯 Key Takeaway

Hypernetworks can unintentionally introduce shortcut learning when conditioned on strong signals.

This observation highlights the need for careful architectural constraints when combining hypernetworks with auxiliary predictions.
